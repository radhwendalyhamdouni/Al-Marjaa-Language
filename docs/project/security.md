1. **الحد الأدنى من الامتيازات:** كل مكون يعمل بأقل امتيازات ممكنة

## 3. نموذج التهديدات

### 3.1 الملفات

| التهديد | الوصف | التخفيف |
|---------|-------|---------|
| File Read | قراءة ملفات غير مصرح بها | sandboxing |
| File Write | كتابة ملفات غير مصرح بها | sandboxing |

### 3.2 الشبكة

| التهديد | الوصف | التخفيف |
|---------|-------|---------|
| SSRF | Server-side request forgery | whitelist |
| Data Exfiltration | تسريب البيانات | sandboxing |

### 3.3 الذاكرة

| التهديد | الوصف | التخفيف |
|---------|-------|---------|
| Memory Leak | تسرب الذاكرة | Rust ownership |
| Buffer Overflow | فيض الذاكرة | Rust safety |

## 10. ضوابط الشبكة في المفسّر (مضاف)

تم إضافة واجهات شبكة آمنة داخل اللغة:
- `http_أحضر_آمن(url، خيارات)`
- `http_أرسل_آمن(url، body، method، خيارات)`

**إجراءات الحماية الافتراضية:**
- منع الوصول إلى المضيفات الداخلية مثل `localhost` وعناوين IP الخاصة/link-local.
- السماح فقط ببروتوكولي `http` و`https`.
- التحقق من مهلة الطلب ضمن مدى آمن (1 إلى 300 ثانية).

**خيارات متقدمة:**
- دعم وكيل شبكة `وكيل`.
- دعم رؤوس HTTP مخصصة `رؤوس`.
- وضع تخفي `تخفي` عبر User-Agent افتراضي أو مخصص `اسم_عميل`.
- كسر القيد الأمني للشبكات الداخلية يتطلب تفعيلًا صريحًا عبر `سماح_شبكة_داخلية`.

> تنبيه: تفعيل `سماح_شبكة_داخلية` قد يزيد مخاطر SSRF إذا استُخدم مع مدخلات غير موثوقة.
