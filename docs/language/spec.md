## 1) الهدف والنطاق

هذه المواصفة تصف السلوك الرسمي للغة المرجع (Al-Marjaa) على ثلاثة مستويات:
1. **Lexical + Syntax**: كيف يُكتب البرنامج ويُحلّل.
2. **Semantics**: كيف تُقيَّم التعابير وتُنفّذ التعليمات.
3. **Edge Cases**: سلوك الحالات الحدّية المتوقعة.

> هذه النسخة تركّز على ما هو مدعوم فعلياً في المفسّر الحالي، لتقليل الفجوة بين التوثيق والتنفيذ.

## 2) القواعد المعجمية (Lexical)

### 2.1 المحارف المدعومة
- العربية: ا-ي (مع الأشكال الشائعة: ة، ؤ، إ، أ، آ).
- اللاتينية: `a-z`, `A-Z`.
- الأرقام: `0-9` و `٠-٩`.
- `_` و`ـ` داخل المعرّفات.

### 2.3 أنواع الرموز
- **Literals:** Number, String, Boolean, Null.
- **Operators:**
  - حسابية: `+ - * / % ^ //`
  - إسناد: `= += -= *= /= %= ^=`
  - مقارنة: `== != < > <= >=`
  - منطقية: `و/&&`, `أو/||`, `ليس/!`
- **Punctuators:** `;`/`؛`, `,`/`،`, `()[]{}`, `.`, `->`.

## 3) القواعد النحوية (Syntax - EBNF مبسّط)

```ebnf
Program        = { Statement } EOF ;

Statement      = VariableDecl
               | FunctionDecl
               | IfStmt
               | WhileStmt
               | ForStmt
               | ReturnStmt
               | BreakStmt
               | ContinueStmt
               | ExpressionStmt
               | Block ;

VariableDecl   = ("متغير" | "م" | "ثابت" | "ث") Identifier ["=" Expression] Terminator ;
FunctionDecl   = "دالة" Identifier "(" [Parameters] ")" Block ;
IfStmt         = "إذا" "(" Expression ")" Statement
                 { "وإذا" "(" Expression ")" Statement }
                 [ "وإلا" Statement ] ;
WhileStmt      = "طالما" "(" Expression ")" Statement ;
ForStmt        = "لكل" Identifier "في" Expression Statement ;
ReturnStmt     = ("أرجع" | "ارجع") [Expression] Terminator ;
BreakStmt      = "توقف" Terminator ;
ContinueStmt   = ("أكمل" | "اكمل") Terminator ;
Block          = "{" { Statement } "}" ;
Terminator     = ";" | "؛" ;

Expression     = Assignment ;
Assignment     = ((Identifier | PropertyAccess) "=" Expression) | Ternary ;
Ternary        = LogicOr ["?" Expression ":" Expression] ;
LogicOr        = LogicAnd { ("أو" | "او" | "||") LogicAnd } ;
LogicAnd       = Comparison { ("و" | "&&") Comparison } ;
Comparison     = Term { ("<" | ">" | "<=" | ">=" | "==" | "!=") Term } ;
Term           = Factor { ("+" | "-") Factor } ;
Factor         = Unary { ("*" | "/" | "%" | "^") Unary } ;
Unary          = (("ليس" | "-") Unary) | Call ;
Call           = Primary { "(" [Arguments] ")" | "." Identifier | "[" Expression "]" } ;
Primary        = Number | String | Boolean | Null | Identifier | List | Dictionary | "(" Expression ")" ;
```

## 4) الدلالات (Semantics)

### 4.1 الأنواع الأساسية
- `Number`, `String`, `Boolean`, `Null`, `List`, `Dictionary`, `Function`, `Class/Instance`.

### 4.2 Truthiness
- `خطأ` و`لا_شيء` قيمتان falsy.
- بقية القيم truthy افتراضياً (مع مراعاة سلوك المفسّر الحالي).

### 4.3 Scoping
- نطاقات متداخلة (lexical scoping).
- الكتلة `{ ... }` تنشئ نطاقاً فرعياً.
- البحث عن المتغيرات من الداخل إلى الخارج.

### 4.4 Mutability
- `ثابت` يمنع **إعادة الإسناد** لنفس المعرّف.
- القوائم/القواميس قابلة للتعديل داخلياً (mutable contents).
- الأنواع العددية/النصية/المنطقية تعامل كقيم غير قابلة للتعديل المباشر.

### 4.5 ترتيب التقييم
- العمليات الحسابية: يسار → يمين حسب أولوية العوامل.
- الإسناد: يمين → يسار.
- المنطق (`و`, `أو`) يدعم short-circuit.

### 4.6 استدعاء الدوال
- تمرير المعاملات بالترتيب.
- إذا غاب `أرجع`، تكون النتيجة `لا_شيء`.

## 5) الحالات الحدّية (Edge Cases)

### 5.1 القسمة على صفر
- سلوك الأعداد العائمة في Rust يُستخدم كمرجع:
  - `10 / 0` قد ينتج `Infinity` أو `-Infinity` حسب الإشارة.
  - `0 / 0` ينتج `NaN`.

### 5.2 NaN
- أي مقارنة مساواة مباشرة مع `NaN` تعتبر غير موثوقة (`NaN != NaN`).
- يجب استخدام دالة مساعدة/تحقق صريح عند الحاجة لتمييزها.

### 5.3 Null
- الوصول لمفتاح غير موجود في قاموس أو فهرس خارج النطاق في قائمة قد يعيد `لا_شيء`.
- العمليات الحسابية المباشرة مع `لا_شيء` يجب أن تعطي خطأ runtime واضح.

### 5.4 Type coercion
- لا يوجد coercion واسع تلقائي على نمط JavaScript.
- التحويلات تكون صريحة عبر دوال المكتبة المدمجة حيث توفّرت.

## 6) أمثلة مرجعية مختصرة

```almarjaa
متغير س = ١٠؛
متغير ص = ٠؛
اطبع(س / ص)؛   # Infinity متوقع وفق IEEE-754
```

```almarjaa
دالة جمع(أ، ب) {
  أرجع أ + ب؛
}
اطبع(جمع(٢، ٣))؛
```

```almarjaa
متغير عناصر = [١، ٢]؛
اطبع(عناصر[١٠])؛  # لا_شيء
```
